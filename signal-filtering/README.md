# RC Filter Signal Generation and Analysis

This project demonstrates how to use a Raspberry Pi to generate square
wave signals and analyze the behavior of low- or high-pass filters
with different frequencies. The main objective is to observe the
smoothing effect of the RC filter on the input square wave at various
frequencies and visualize the results.

## Overview

The project uses the Raspberry Pi's GPIO pins, an MCP3008 ADC, and a
circuit consisting of a resistor (R) and a capacitor (C). The square
wave signal is generated by the Raspberry Pi, and the filtered output
is read through the MCP3008. The results are plotted using **gnuplot**
to visualize the filtering effect.

### Key Components:
- **Raspberry Pi (GPIO pins)**: Used for generating square wave signals and reading ADC values.
- **MCP3008**: An 8-channel ADC used to read the filtered signal.
- **Resistor (R)**: 10kΩ.
- **Capacitor (C)**: 100nF.

## Setup Instructions

### Hardware Connections

1. **RC Circuit**:
   - Connect the **10kΩ resistor** in series with the **100nF capacitor**.

   ***Low-pass-filter***

   - Connect one end of the resistor to **GPIO37 (BCM GPIO26)** and the other end to both the **MCP3008 channel 2 (pin 3)** and the capacitor.
   - Connect the other end of the capacitor to the **ground (GPIO39)**.

   ***High-pass-filter***

   - Connect one end of the capacitor to **GPIO37 (BCM GPIO26)** and the other end to both the **MCP3008 channel 2 (pin 3)** and the resistor.
   - Connect the other end of the resistor to the **ground (GPIO39)**.

2. **Raspberry Pi GPIO**:
   - Set **GPIO37 (BCM GPIO26)** as the output for the square wave.
   - Set **GPIO39** to ground.
   - Use the **MCP3008** for ADC readings on **Channel 2**.
   - Make sure the **MCP3008** is correctly connected to the Raspberry Pi SPI pins (CE0, MOSI, MISO, and SCK).


### Software Setup

1. **Install the bcm2835 library**:
   If you haven't already, install the `bcm2835` library to control the GPIO and SPI communication on the Raspberry Pi:
   ```
   sudo apt-get install libbcm2835-dev
   ```

2. **Compile and Run the Program**:

The program to generate a square wave signal and read ADC values.
Compile the program by typing "make" and run the program as super user
with "sudo ./rc01".

### Graphical Representation

The results can be visualized by plotting the filtered signal and comparing 
it with the input signal (square wave). You can use gnuplot to plot the ADC
readings and the square wave signal over time.
To generate the ADC vs. time data for the plot, follow these steps:

1. Capture ADC data using the following command:

```
sudo ./rc01 > output.txt
```

2. Process the output with awk to extract the relevant data for plotting
(time and ADC readings):

```
awk '{print NR*5-5, $5}' output.txt > signals_vs_time.txt
```

This works since the raspberry is sampling ADC values every 5 ms, and this
command formats the data to include time (in ms) and the corresponding
ADC value.

3. Generate the plot using gnuplot:

You can use the included "signal.gp" file in the results/ directory
to generate the plot. Just adjust the filenames in the script accordingly.

## Conclusion

This project provides hands-on experience with signal processing and 
filtering using a Raspberry Pi and simple electronic components. 
By adjusting the frequency of the input signal, you can observe how
the RC low-pass or high-pass filter affects the signal as the frequency
increases.

As the frequency of the input signal changes, the high-pass filter 
selectively allows higher frequencies to pass through, while attenuating 
lower frequencies, providing insight into how frequency-dependent filters 
modify waveforms.

Feel free to change the frequency generation and analysis procedure in
the code as you like when experimenting with
different resistor and capacitor values.
